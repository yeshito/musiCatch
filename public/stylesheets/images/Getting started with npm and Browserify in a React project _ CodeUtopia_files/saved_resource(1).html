                    Jani Hartikainen Admin Simple Better • 5 months ago

                    React wants you to add a `key` attribute when you have dynamic children. This helps React figure out when it needs to update the DOM for that.

                    We're rendering a list of ChatMessages here, so to fix this we'd need to include <chatmessage key="{someAttribute}" ...="">. However, right now we don't have a value that would work well as a *unique* key. We're going to add a timestamp to each message later, which we can use to address this :)
